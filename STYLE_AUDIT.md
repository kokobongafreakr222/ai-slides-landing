## Детали по проверкам

### Горизонтальный скролл
- **320px:** `body.scrollWidth === 320` — нет горизонтального скролла.
- **375px:** `body.scrollWidth === 375` — нет горизонтального скролла.
- **768px:** `body.scrollWidth === 768` — нет горизонтального скролла.
- Pill-теги (стили: Минимализм, Corporate, Storytelling…) находятся в контейнере `overflow-x: auto; flex-wrap: nowrap` — скроллятся горизонтально внутри блока, body не переполняют. PASS.

### Header sticky
- `position: fixed; top: 0px; z-index: 300` — sticky работает корректно на всех breakpoints.
- На 320px и 375px десктопная навигация скрыта, в header остаются только логотип DeckMind и CTA-кнопка. Корректное мобильное поведение.

### Контрастность текста (WCAG AA — минимум 4.5:1)

| Элемент | Цвет текста | Фон | Ratio | Статус |
|---------|-------------|-----|-------|--------|
| h1 | rgb(232,232,240) | rgb(10,10,15) | **16.21** | PASS |
| h2 | rgb(232,232,240) | rgb(17,17,24) | **15.42** | PASS |
| nav links | rgb(232,232,240) | rgba(10,10,15,0.85) | **16.21** | PASS |
| paragraph | rgb(232,232,240) | rgb(10,10,15) | **16.21** | PASS |
| **CTA кнопка** | rgb(255,255,255) | rgb(91,106,240) | **4.41** | **FAIL** |
| **Footer copyright** | rgb(74,74,94) | rgb(10,10,15) | **2.29** | **FAIL** |

**FAIL #1 — CTA-кнопка:** белый текст на фиолетовом `#5B6AF0` даёт 4.41:1 — ниже порога WCAG AA (4.5:1). Нужно либо затемнить фон до ~`#5460D8`, либо использовать более светлый текст.

**FAIL #2 — Footer copyright `© 2025 DeckMind`:** тёмно-серый `rgb(74,74,94)` на почти-чёрном фоне даёт 2.29:1 — критически низкий контраст. Нужно осветлить текст минимум до `rgb(130,130,155)` для достижения 4.5:1.

### Touch targets (минимум 44×44px по WCAG 2.5.5)

Элементы с недостаточным размером touch target:

| Элемент | Ширина | Высота | Проблема |
|---------|--------|--------|----------|
| Логотип "DeckMind" (header) | 77px | 24px | Высота 24px < 44px |
| Логотип "DeckMind" (375px) | 70px | 22px | Высота 22px < 44px |
| "Как работает" (nav, 1440px) | 104px | 24px | Высота 24px < 44px |
| "Кейсы" (nav, 1440px) | 50px | 24px | Высота 24px < 44px |
| "Вопросы" (nav, 1440px) | 70px | 24px | Высота 24px < 44px |
| Footer links | 131–247px | 22–24px | Высота < 44px |

Навигационные ссылки в header на мобильном (320/375) скрыты — WCAG не применим. Проблема актуальна для логотипа и footer-ссылок на всех размерах, и для nav-ссылок на десктопе.

**Рекомендация:** добавить `min-height: 44px` + `flex items-center` к nav-ссылкам и footer-ссылкам.

### ARIA и семантика

- Accordion (FAQ): все 7 кнопок имеют `aria-expanded` (true/false) и `aria-controls` с корректным ID. PASS.
- Кнопки без aria-expanded: не обнаружены — все button-элементы в accordion.
- Ссылка "Перейти к содержимому" (skip-link): присутствует, ведёт на `#main-content`. Физический размер 1×1px — скрытая, корректно. PASS.
- `<header>` → `banner`, `<main>`, `<footer>` → `contentinfo` — семантические роли расставлены. PASS.
- `<nav aria-label="Основная навигация">`, `<nav aria-label="Навигация футера">` — PASS.

### CTA видимость

- Hero: кнопка "Создать презентацию" видна на всех breakpoints.
- Footer CTA: кнопка "Создать презентацию" видна.
- На 320px CTA в header компактная, не обрезана.
